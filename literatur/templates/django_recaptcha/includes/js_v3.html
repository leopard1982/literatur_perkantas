{% comment %} Override of django_recaptcha's default template: the vendored version uses a global `element` variable and a global `recaptchaFormSubmit` function name. When more than one reCAPTCHA v3 widget is rendered on the same page (e.g. Login + Register + Reset Password modals), each widget's script overwrites those globals, so whichever widget's script runs last "wins" and hijacks every other form's submit handler. Wrapping in an IIFE keeps each widget's state and handler scoped to itself. {% endcomment %}
